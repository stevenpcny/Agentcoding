# Goal-Driven Workflows

## Core rule
Complex work benefits from being driven by explicit goals rather than open-ended instructions. This playbook serves as an umbrella for goal-oriented agent execution patterns.

## Industry Alignment
Reflects goal-oriented agent design patterns used in production systems and aligns with structured workflow thinking from project management and software engineering.

## Scope
This is an umbrella playbook. For detailed guidance, refer to the more specific playbooks:

- Goal and Task Framing
- Verification and Review Loops
- Self-Correction and Recovery
- Testing Strategy
- Large-Scale Refactoring

## Key Principle
Always start with a clear goal. Break work into verifiable phases. Use the more specific playbooks for execution details.

## When to Reference This Playbook
- When designing high-level agent workflows
- When other specific playbooks do not fully cover the task pattern
- As an entry point for understanding goal-driven agent behavior

## Recommendation
Prefer the more specific playbooks listed above for day-to-day use. This file provides the overarching philosophy.
